[1] Deleted old monolith src/index.ts. Created src/types.ts (Provider interface + CompleteOptions), src/schemas.ts (4 Zod schemas: AskResponse, ReviewResponse, VerifyResponse, CompareResponse), src/providers/ and src/tools/ dirs | zod is transitive from MCP SDK, no extra install needed. Schemas match CLAUDE.md spec exactly with enums for verdict/confidence/severity fields.
[2] Created src/structured-output.ts with getStructuredResponse<T>(raw, schema, retryFn?) pipeline: JSON.parse → retry via callback → regex extract first{..last} → Zod safeParse. Exports StructuredError type and isStructuredError guard. | Never throw pattern: every failure path returns {verdict:"ERROR", error:string}. Regex uses indexOf/lastIndexOf instead of greedy regex for reliability. Zod error issues formatted with path for debuggability. retryFn is optional async callback so the pipeline stays provider-agnostic.
[3] Created src/providers/gemini.ts implementing Provider interface with @google/genai SDK. Uses ai.models.generateContent() with config object (systemInstruction, temperature, maxOutputTokens). 30s timeout via Promise.race. | @google/genai API: GoogleGenAI class, models.generateContent() returns object with .text property. Config uses systemInstruction (not systemPrompt), maxOutputTokens (not maxTokens). Context7 confirmed current API shape.
[4] Created src/providers/openai-compatible.ts implementing Provider interface with openai npm package. Configurable baseURL for OpenAI/Groq/DeepSeek. Uses chat.completions.create with response_format json_object. 30s timeout via Promise.race. Installed openai dependency. | openai SDK: new OpenAI({apiKey, baseURL}) for custom endpoints. chat.completions.create returns choices[0].message.content. response_format: {type:"json_object"} forces JSON mode. Constructor takes name param so error messages identify the provider.
